User Permission Denied — Bundle by IP

Bundle permission denied events by source IP over 24 hours. If volume exceeds 6, raise a ticket for investigation — could indicate privilege escalation, misconfigured app, or a user hitting walls they shouldn't be.

What triggers this rule

  • Event category is user_permission_denied
  • Bundles every event from the same source IP for 24 hours
  • Fires after 6 events accumulate
  • Fires once per bundle (no re-trigger)

What happens when it fires

  • Sends an in-app + push notification to anyone with manage_security
  • No LLM triage configured — add one
