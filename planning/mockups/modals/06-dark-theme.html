// mockup 06 / dark theme

Dark Theme.

Same merged direction as Mockup 05, ported to dark. Brighter accent tones (lighter blue / green / amber / red) read better against the dark surface. Tint bumped from 5% → 10% because dark backgrounds absorb color — the same percentage that felt subtle on white would be invisible here. Accent buttons use a dark text color (against the bright accent fill) for AAA contrast. Inputs are dark-on-dark with a stronger border so they don't disappear into the tinted card.

// dark theme — the rules I'd ship

Brighter accent palette in dark — `#60a5fa` not `#2563eb`, etc. The Bootstrap defaults read too dark on a dark surface. Bump the tint percentage — 5% on white ≈ 10% on dark. Accent button text goes dark on the bright accent fill for AAA contrast (white-on-light-blue fails). Inputs stay dark-on-dark with a stronger border so they don't disappear into the tinted card surface. Triggered by `prefers-color-scheme: dark` in the production CSS, with the same rules for `data-bs-theme="dark"`.
